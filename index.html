Overview

This is the build generated documentation of the SOLAR solutions architecture framework developed to provide common feature sets and building processes each and every web based application requires as part of the standard development process. The content presented here is self-generated and describes the current feature set of the solution as at build-time.

The framework/platform is a generic solution for web based applications development where fundamental implementation is abstracted and inherited by future projects so as to remove the repeated duplication of effort required to generate a given web-based application.

This workspace provides information about Health Canada's Common Platform. The platform was originally designed in Oct 2011 and went into production in support of the RARM/Kina application in Nov 2012.

Developer details

See the Developer Details page for instructions on how to set things up.

Application Business Requirements Docs (Fall 2010)

The common platform was developed based on the requirements expressed by the following three business lines:

  • RARM (Kina) BRD
  • Drug Disposal BRD
  • Dosimetry BRD

Application utilization

The following is a list of applications that have embraced and fully support the rational of the Platform:

  • Secure Gateway
  • Secure Gateway Business Management App
  • Secure Gateway On-Boarding Administration Application
  • Health Canada common shared services instances
    • Code Service
    • Resource Service
    • User Enrollment Service
    • Data Entry Framework Service
  • Administrative applications to support the above services
    • Code Service Admin App
    • Resource Service Admin App
    • User Service Admin App
    • Data Entry Service Admin App

This effort is of demo quality and is work-in-progress toward a production instance.

Developers may suggest suggest alternative/ addition plugins and other such commentary. The content generated is from specific branch/tag or head as the application reports.
