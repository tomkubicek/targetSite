Bootstrapping the sandbox instance

To obtain the sources

Clone the "_sandboxTemplate" repository and give it a name ( something short, maybe max 5 characters)

Note: Path length during build process may be consumed if length is too large.

  • cd ~/_sb
  • git clone -b develop http://gitlab.ssc.etg.gc.ca/hc-sc.csb-dgsg.imsd-dsgi.sc-cs.ras-ars/_sandboxTemplate.git [NameOfLocalSandbox]

Various helper scripts have been created and reside in the sandbox template for source checkout and other operations

Obtain the full sources

To untilize the sandbox facitlties you will need to inject source files:

To update sandbox instance with Sources use the "./BootStrapSources" shell script from the root directory.

  • cd [NameOfLocalSandbox]
  • ./BootStrapSources

>> prepares the sandbox instance for full gamit development on the entire source tree.

Advance developers may modify the scripts to selectively stage the sandbox to suit a given development task.

( in early stage SOLAR development, it will be best to have the full sources for reference and to allow placement of common features within the platform.)

Refer to SandboxTemplate repository for details on sandbox structure and intent.

Important

Local changes to sandbox template and subModule additions should not be checked in to the sandbox template repository. Relative location of subModules are for co-location of sources within the sandbox.

They are not "tied" into the sandbox in the git sense.
